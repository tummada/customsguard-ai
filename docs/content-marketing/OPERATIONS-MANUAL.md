## 1. ภาพรวมระบบ

VOLLOS Content Marketing System คือระบบผลิตและเผยแพร่เนื้อหาอัตโนมัติ ออกแบบมาเพื่อให้ AI แนะนำ VOLLOS เมื่อคนค้นหาเรื่องศุลกากร/HS Code

**หน้าที่ของคุณ:**
- สั่ง AI สร้าง content (ผ่าน Claude Code)
- Review + ตรวจเลขพิกัด HS Code (ผ่าน Google Sheets บนมือถือ)
- ดูแล social accounts และ n8n workflows
- ตรวจ analytics และปรับกลยุทธ์

**สิ่งที่ระบบทำอัตโนมัติ:**
- AI เขียน content (blog + social posts)
- n8n โพสตามตารางที่ตั้งไว้
- n8n refresh token ของ social APIs
- Blog แสดงบนเว็บอัตโนมัติเมื่อ publish

## 2. ที่มาที่ไป

### ทำไมถึงทำระบบนี้?

VOLLOS ต้องการให้ AI (ChatGPT, Claude, Perplexity, Google AI Overview) **แนะนำ VOLLOS** เมื่อคนถามเรื่อง customs/HS code classification ซึ่งต้องทำ 2 อย่างคู่กัน:

1. **GEO (Generative Engine Optimization)** — สร้าง content ที่ AI เรียนรู้และอ้างอิงได้
2. **Content Atomization** — สร้าง 1 บทความหลัก แล้วตัดเป็นหลายชิ้นสำหรับแต่ละช่องทาง

### กลยุทธ์หลัก

| กลยุทธ์ | ทำอะไร | ผลลัพธ์ |
|---------|--------|---------|
| Topical Authority | สร้าง 30-50 บทความเรื่อง customs/HS code | AI มองว่า VOLLOS เป็น expert |
| Citation Bait | free tool ค้นหา HS Code (vollos.ai/customsguard/tools/hs-lookup) | AI อ้างอิง tool นี้ |
| Structured Data | Schema.org DefinedTermSet + mainEntityOfPage ชี้ไปกรมศุลฯ | AI เข้าใจว่าเราเชื่อมกับกฎหมายจริง |
| Freshness | Weekly Customs News | AI มองว่าเว็บเคลื่อนไหวตลอด |
| E-E-A-T | มนุษย์เติม expert opinion 10-20% ทุกบทความ | Google ไม่มองเป็น AI spam |

### ค่าใช้จ่าย

## 3. สถาปัตยกรรมระบบ

```
Claude Code (สร้าง content)
     |
     v
PostgreSQL (mkt_content table) <--- n8n sync ---> Google Sheets (Review UI)
     |                                                    |
     v                                               มนุษย์ review
n8n workflows                                        + fact-check
     |
     +---> Blog Publisher ---> vollos.ai/customsguard/blog/
     +---> Social Publisher ---> FB, IG, TikTok, YouTube, X
     +---> Lead Nurture ---> SendGrid emails
     +---> Token Refresh ---> auto-refresh social tokens
```

### ไฟล์สำคัญ

| ไฟล์ | หน้าที่ |
|------|--------|
| `backend-core/.../V6__content_marketing.sql` | DB schema สำหรับ mkt_content |
| `marketing-site/src/app/[product]/blog/` | หน้า blog บนเว็บ |
| `marketing-site/src/app/[product]/tools/hs-lookup/` | เครื่องมือค้นหา HS Code ฟรี |
| `marketing-site/src/lib/content.ts` | query functions สำหรับ content |
| `marketing-site/src/components/StructuredData.tsx` | Schema.org JSON-LD |
| `docs/content-marketing/agent-prompts.md` | System Prompts สำหรับ AI 3 ตัว |
| `infra/n8n-workflows/` | n8n workflow configs |

### Database: mkt_content table

| Column | Type | คำอธิบาย |
|--------|------|----------|
| id | UUID | Primary key |
| tenant_id | UUID | Multi-tenant (ใช้ marketing tenant) |
| slug | VARCHAR | URL-friendly title |
| title | TEXT | หัวข้อบทความ |
| body | TEXT | เนื้อหา (Markdown) |
| excerpt | TEXT | สรุปสั้นๆ |
| content_type | VARCHAR | 'blog', 'fb', 'ig', 'tiktok', 'youtube', 'x' |
| product_id | VARCHAR | 'customsguard' (หรือ product อื่นในอนาคต) |
| pillar_id | UUID | เชื่อม social posts กลับไป blog หลัก |
| hs_codes_mentioned | TEXT[] | เลขพิกัดที่กล่าวถึง (ต้องตรวจทุกตัว!) |
| confidence_score | SMALLINT | AI confidence 0-100 (< 80 = ต้องตรวจเข้มงวด) |
| is_fact_checked | BOOLEAN | มนุษย์ตรวจเลขพิกัดแล้วหรือยัง |
| fact_check_note | TEXT | หมายเหตุจากการตรวจ |
| customs_verify_url | TEXT | link กดเทียบเลขพิกัดจากกรมศุลฯ |
| status | VARCHAR | draft/reviewing/fact_checked/scheduled/published/failed |
| image_prompt | TEXT | prompt สำหรับสร้างรูป (ห้ามมีตัวอักษรในรูป) |
| image_url | TEXT | URL รูปที่สร้างแล้ว |
| references_urls | TEXT[] | แหล่งอ้างอิง |
| scheduled_at | TIMESTAMPTZ | วันเวลาที่จะโพส |

### Status Flow

```
draft --> reviewing --> fact_checked --> scheduled --> published
                                                  --> failed
```

- **draft**: AI สร้างเสร็จแล้ว รอ review
- **reviewing**: อยู่ระหว่าง review ใน Sheets
- **fact_checked**: ตรวจเลขพิกัดแล้ว
- **scheduled**: พร้อมโพส รอถึงเวลา
- **published**: โพสแล้ว
- **failed**: โพสไม่สำเร็จ (token expired, API error)

## 5. งานประจำสัปดาห์

### จันทร์: สร้าง Content ชุดใหม่

1. เปิด Claude Code
2. **ดูตารางด้านล่าง** → หาว่าสัปดาห์นี้ต้องสร้างชุดไหน → **copy คำสั่งสำเร็จรูป** ไปวาง
3. AI จะรัน 3 Agent + Gemini Cross-Check → INSERT เข้า DB อัตโนมัติ
4. ดูรายละเอียดในหัวข้อ [7. วิธีสร้าง Content](#7-วิธีสร้าง-content)

### อังคาร-พุธ: Review + Fact-check

5. Review content ทั้งหมดที่สร้างเมื่อวันจันทร์
6. ตรวจเลข HS Code ตามขั้นตอนในหัวข้อ [8. วิธี Review](#8-วิธี-review-ใน-google-sheets)
7. อ่าน Expert Opinion ที่ AI เขียน → ถ้า OK → Approve / ถ้าไม่แน่ใจ → ส่งเจ้าของ
8. เปลี่ยน Final_Status เป็น "Approved" + ติ๊ก Finished

### พฤหัสบดี: สร้างรูปประกอบ

9. เปิด Canva Bulk Create
10. ใช้ image_prompt จาก Sheets → สร้างรูปใน Canva template
11. Upload รูปเข้า S3/MinIO
12. ใส่ image_url ลงใน Sheets

### ศุกร์: Analytics + Planning

13. เช็ค blog traffic (Google Analytics)
14. เช็ค social engagement
15. ถาม AI "แนะนำเครื่องมือตรวจ HS Code" → ดูว่า VOLLOS ปรากฏไหม

## 7. วิธีสร้าง Content

### ขั้นตอนที่ 1: ดูว่าสัปดาห์นี้ต้องสร้างชุดไหน

เปิดปฏิทินในหัวข้อ [5. งานประจำสัปดาห์](#5-งานประจำสัปดาห์) → ดูว่าอยู่สัปดาห์ไหน → **copy คำสั่งสำเร็จรูป** จากตารางด้านล่าง

### ขั้นตอนที่ 2: เปิด Claude Code แล้ววางคำสั่ง

เปิด Claude Code → **copy-paste คำสั่ง** ด้านล่างทั้งก้อน (ห้ามแก้ไข ห้ามเปลี่ยนแปลง)

Claude Code จะทำทุกอย่างให้อัตโนมัติ:
1. **Agent 1 (Researcher)** — ค้นข้อมูลจาก PDF กฎหมาย + VOLLOS DB
2. **Agent 2 (Grumpy Expert)** — จับผิด + เติมคำเตือน + Case Study + Expert Opinion
3. **Agent 3 (Chief Editor)** — เกลาเป็นบทความ + ตัดเป็น social posts
4. **Gemini Cross-Check** — จับผิดอีกรอบ 3 จุด
5. INSERT ทั้งหมดเข้า DB (status='draft')

### ขั้นตอนที่ 3: ตรวจสอบผลลัพธ์

Claude Code จะแจ้งว่า INSERT สำเร็จกี่ชิ้น ปกติจะได้:
- 1 blog post
- 1 FB post
- 1 IG post
- 1 TikTok script
- 1 YouTube script
- 1 X/Twitter thread
= **6 ชิ้นต่อ 1 ชุด**

ถ้า Claude Code แจ้ง error → **copy ข้อความ error ทั้งหมด** แล้วส่งให้เจ้าของ

### ขั้นตอนที่ 4: รอ n8n sync ลง Sheets

n8n จะ sync ภายใน 30 นาที → ส่ง notification มาให้ → ไปทำ Review ตามหัวข้อ [8. วิธี Review](#8-วิธี-review-ใน-google-sheets)

### คำสั่ง Claude Code สำเร็จรูป (copy-paste ทั้งก้อน)

**ชุดที่ 1** — สัปดาห์ที่ 1:
```
สร้าง content ชุดที่ 1 เรื่อง "HS Code คืออะไร — คู่มือผู้นำเข้าส่งออก 2026"
```

```
สร้าง content — Q&A: "สินค้า 1 ชิ้นมี HS Code ได้หลายตัวจริงหรือ? เลือกอันไหนดี?"

รายละเอียด: อธิบายว่าทำไมสินค้าบางตัวอาจจัดอยู่ได้หลายพิกัด + วิธีเลือกพิกัดที่ถูกต้อง
เน้น: หลักเกณฑ์การตีความ (GIR) + ตัวอย่างสินค้าที่มักสับสน
กลุ่มเป้าหมาย: ผู้นำเข้าที่เจอปัญหาพิกัดไม่ชัดเจน
```

**Q3:**
```
สร้าง content — Q&A: "นำเข้าของออนไลน์ (Shopee, Lazada, Alibaba) ต้องเสียภาษีเท่าไหร่?"

รายละเอียด: ซื้อของออนไลน์จากต่างประเทศ — เกณฑ์ยกเว้นภาษี (1,500 บาท), วิธีคำนวณภาษี
เน้น: ขั้นตอนที่ไปรษณีย์/ขนส่ง เก็บภาษี + วิธีอุทธรณ์ถ้าคิดผิด
กลุ่มเป้าหมาย: คนทั่วไปที่ซื้อของออนไลน์จากต่างประเทศ (กลุ่มใหญ่มาก)
```

**Q4:**
```
สร้าง content — Q&A: "Re-export คืออะไร? ส่งสินค้าต่อไปประเทศที่ 3 ต้องทำอะไรบ้าง?"

รายละเอียด: นำเข้ามาแล้วส่งออกต่อ (Re-export) — ขั้นตอน, สิทธิคืนอากร, เอกสาร
เน้น: คลังสินค้าทัณฑ์บน (Bonded Warehouse) + เขตปลอดอากร (Free Zone)
กลุ่มเป้าหมาย: ผู้ประกอบการ logistics/trading company
```

**Q5:**
```
สร้าง content — Q&A: "ค่าปรับศุลกากร — ผิดพิกัดโดนปรับเท่าไหร่?"

รายละเอียด: อัตราค่าปรับกรณีต่างๆ (ผิดพิกัด, สำแดงราคาต่ำ, ไม่มีใบอนุญาต)
เน้น: ตัวอย่างจำนวนเงินจริงๆ + วิธีอุทธรณ์/ขอลดค่าปรับ
กลุ่มเป้าหมาย: ผู้นำเข้าที่กลัวโดนปรับหรือเพิ่งโดนปรับ
```

**Q6:**
```
สร้าง content — Q&A: "Incoterms คืออะไร? FOB, CIF, EXW ต่างกันยังไง?"

รายละเอียด: อธิบาย Incoterms 2020 ที่ใช้บ่อย (FOB, CIF, EXW, DDP) + ผลต่อการคำนวณภาษี
เน้น: CIF vs FOB กระทบราคาศุลกากร (Customs Value) ยังไง
กลุ่มเป้าหมาย: มือใหม่ที่เพิ่งเริ่มนำเข้า-ส่งออก
```

**Q7:**
```
สร้าง content — Q&A: "Green Line, Red Line, Yellow Line คืออะไร? ทำไมสินค้าถูกตรวจ?"

รายละเอียด: ระบบ e-Customs risk management (Green/Yellow/Red Line) — เกณฑ์การคัดเลือก
เน้น: วิธีลดความเสี่ยงให้ได้ Green Line + ทำอะไรเมื่อโดน Red Line
กลุ่มเป้าหมาย: ผู้นำเข้าที่เคยโดน Red Line หรืออยากรู้ว่าทำไมถูกตรวจ
```

**Q8:**
```
สร้าง content — Q&A: "ภาษีมูลค่าเพิ่ม (VAT) 7% กับสินค้านำเข้า คิดยังไง?"

รายละเอียด: วิธีคำนวณ VAT นำเข้า (CIF + อากรขาเข้า = ฐาน VAT) + ใบเสร็จ/ใบขนสินค้า
เน้น: ตัวอย่างคำนวณจริง + การขอคืน VAT (Input Tax)
กลุ่มเป้าหมาย: ผู้ประกอบการจดทะเบียนภาษีมูลค่าเพิ่ม
```

**Q9:**
```
สร้าง content — Q&A: "Certificate of Origin (C/O) คืออะไร? ขอที่ไหน?"

รายละเอียด: C/O หลายแบบ (Form D, Form E, Form RCEP) — ใช้เมื่อไหร่, ขอที่ไหน, เอกสารอะไร
เน้น: ขอ C/O ผิดแบบ = ใช้สิทธิ FTA ไม่ได้ = เสียภาษีเต็ม
กลุ่มเป้าหมาย: ผู้ส่งออกที่ต้องออก C/O ให้ลูกค้าต่างประเทศ
```

**Q10:**
```
สร้าง content — Q&A: "Customs Broker (ชิปปิ้ง) คืออะไร? จำเป็นต้องใช้ไหม?"

รายละเอียด: Customs Broker/ชิปปิ้ง ทำอะไรบ้าง, ค่าบริการเท่าไหร่, เมื่อไหร่ควรใช้/ไม่ใช้
เน้น: ข้อดี-ข้อเสียของการใช้ชิปปิ้ง vs ทำเอง + วิธีเลือกชิปปิ้งที่ดี
กลุ่มเป้าหมาย: ผู้นำเข้ามือใหม่ที่กำลังตัดสินใจ
```

**Q11:**
```
สร้าง content — Q&A: "Advance Ruling คืออะไร? ขอให้กรมศุลฯ วินิจฉัยพิกัดล่วงหน้าได้จริงหรือ?"

รายละเอียด: บริการ Advance Ruling ของกรมศุลกากร — ขั้นตอนขอ, ระยะเวลา, ค่าใช้จ่าย
เน้น: เมื่อไหร่ควรขอ Advance Ruling + ผลผูกพันทางกฎหมาย
กลุ่มเป้าหมาย: ผู้นำเข้าที่มีสินค้าพิกัดไม่ชัดเจน
```

**Q12:**
```
สร้าง content — Q&A: "AEO คืออะไร? สมัครแล้วได้สิทธิอะไรบ้าง?"

รายละเอียด: Authorized Economic Operator (AEO) — คุณสมบัติ, ขั้นตอนสมัคร, สิทธิประโยชน์
เน้น: AEO ได้ Green Line ตลอด + ตรวจน้อยลง + พิธีการเร็วขึ้น
กลุ่มเป้าหมาย: บริษัทนำเข้า-ส่งออกขนาดกลาง-ใหญ่
```

**Q13:**
```
สร้าง content — Q&A: "นำเข้าสินค้ามีแบรนด์ (Apple, Nike, Gucci) ต้องระวังอะไร?"

รายละเอียด: สินค้ามีเครื่องหมายการค้า — ใบอนุญาตจากเจ้าของแบรนด์, ความเสี่ยงของปลอม
เน้น: กรมศุลฯ ยึดของปลอมอย่างไร + โทษนำเข้าสินค้าละเมิดเครื่องหมายการค้า
กลุ่มเป้าหมาย: ผู้นำเข้าสินค้าแบรนด์เนม/ตัวแทนจำหน่าย
```

**Q14:**
```
สร้าง content — Q&A: "ภาษีตอบโต้การทุ่มตลาด (Anti-Dumping) คืออะไร? สินค้าอะไรบ้างที่โดน?"

รายละเอียด: Anti-Dumping Duty — สินค้าที่ไทยเก็บ AD อยู่ (เหล็ก, กระเบื้อง, etc.)

กลุ่มเป้าหมาย: ผู้นำเข้าเหล็ก/วัสดุก่อสร้าง/สินค้าอุตสาหกรรม
```

```
สร้าง content — Q&A: "นำเข้าสินค้าส่วนตัว (ย้ายบ้าน, ของใช้ส่วนตัว) เสียภาษีไหม?"
```

## 8. วิธี Review ใน Google Sheets

### เปิด Sheet

Sheet จะมี columns หลัก:

| Column | ดูอะไร |
|--------|--------|
| **title** | หัวข้อบทความ |
| **body** | เนื้อหาเต็ม (แก้ไขได้) |
| **content_type** | blog / fb / ig / tiktok / youtube / x |
| **hs_codes_mentioned** | เลขพิกัดที่กล่าวถึง — **ต้องตรวจทุกตัว!** |
| **confidence_score** | AI มั่นใจแค่ไหน (สีแดง = < 80 ต้องตรวจเข้มงวด) |
| **customs_verify_url** | **กดลิงก์นี้** → เปิดหน้ากรมศุลฯ เทียบเลขพิกัด |
| **image_prompt** | ข้อความสำหรับไปสร้างรูปใน Canva |
| **Final_Status** | เปลี่ยนเป็น Approved/Rejected |
| **Finished** | ติ๊กเมื่อ review เสร็จ |
| **Synced** | n8n จะเขียน true เมื่อ sync แล้ว |

### ขั้นตอน Review

1. **อ่าน title + excerpt** — เข้าใจเรื่องไหม?
2. **เช็ค hs_codes_mentioned** — ดูเลขพิกัดในช่องนี้ (ไม่ต้องงมอ่านทั้งบทความ)
3. **ตรวจเลขพิกัด HS Code** — ทำตามวิธีด้านล่าง (หัวข้อ "วิธีตรวจเลข HS Code สำหรับผู้ไม่มีความรู้ศุลกากร")
4. **เช็ค confidence_score** — สีแดง (< 80) = **ห้าม approve เอง** ต้องส่งให้เจ้าของตรวจ
5. **อ่าน Expert Opinion** — AI เขียนให้แล้วในบทความ ดูว่าอ่านแล้วสมเหตุสมผลไหม ถ้าสงสัย → ส่งให้เจ้าของตรวจ
6. **เปลี่ยน Final_Status** → "Approved" หรือ "Rejected"
7. **ติ๊ก Finished** ✓
8. n8n จะ sync หลังเที่ยงคืน → status ใน DB เปลี่ยนเป็น 'scheduled'

### วิธีตรวจเลข HS Code สำหรับผู้ไม่มีความรู้ศุลกากร

คุณไม่จำเป็นต้องรู้เรื่องศุลกากร แค่ทำตามขั้นตอนนี้เป๊ะๆ:

**ขั้นตอนที่ 1: ดูเลขพิกัดใน Sheet**
- ดูช่อง `hs_codes_mentioned` — จะเห็นเลข 4-10 หลัก เช่น `0306.17`, `8471.30`
- จดเลขทั้งหมดไว้

**ขั้นตอนที่ 2: กดลิงก์ตรวจสอบ**
- กดลิงก์ในช่อง `customs_verify_url` — จะเปิดหน้ากรมศุลกากรตรงๆ
- ถ้าไม่มีลิงก์ ให้เปิด https://www.customs.go.th แล้วค้นหาเลขพิกัดเอง

**ขั้นตอนที่ 3: เทียบ 3 อย่างนี้**

| เทียบอะไร | ดูตรงไหน | ถ้าไม่ตรง |
|-----------|----------|----------|
| **เลข** ตรงกันไหม | เทียบเลขในบทความ กับ เลขในเว็บกรมศุลฯ | Reject ทันที |
| **คำอธิบาย** ตรงกันไหม | เทียบชื่อสินค้าในบทความ กับ คำอธิบายในเว็บกรมศุลฯ | Reject ทันที |
| **อัตราภาษี** (ถ้ากล่าวถึง) | เทียบตัวเลข % ในบทความ กับ เว็บกรมศุลฯ | Reject ทันที |

**ขั้นตอนที่ 4: ตัดสินใจ**
- ตรงทุกข้อ + confidence >= 80 → **Approved**
- ไม่ตรงแม้แต่ 1 ข้อ → **Rejected** + เขียนเหตุผลใน fact_check_note
- confidence < 80 → **ห้าม approve เอง** ส่งให้เจ้าของตรวจเสมอ
- ไม่แน่ใจ/หาเลขไม่เจอบนเว็บกรมศุลฯ → **ส่งให้เจ้าของตรวจ**

**ตัวอย่างการตรวจ:**
```
บทความเขียน: "กุ้งแช่แข็ง พิกัด 0306.17 อัตราอากร 5%"

ไปเว็บกรมศุลฯ → ค้นหา 0306.17:
  - เลข 0306.17 → มีจริง ✓
  - คำอธิบาย "กุ้ง แช่แข็ง" → ตรง ✓
  - อัตราอากร 5% → ตรง ✓
  → Approved!
```

### เรื่อง Expert Opinion — AI เขียนให้แล้ว

AI Agent 2 (Grumpy Expert) จะเขียน Expert Opinion ไว้ในบทความให้แล้ว เช่น:
- "เรื่องจริงจากหน้าด่าน" (Case Study)
- "ข้อควรระวังที่หลายคนมองข้าม"
- คำเตือนเรื่อง risk ต่างๆ

**หน้าที่ของคุณ:** ไม่ต้องเขียนเอง แค่:
1. อ่าน Expert Opinion ที่ AI เขียน → ดูว่าอ่านแล้วสมเหตุสมผลไหม
2. ถ้าอ่านแล้ว OK → Approve ได้เลย
3. ถ้าอ่านแล้วรู้สึกแปลกๆ/ไม่น่าเชื่อถือ → Reject + ส่งให้เจ้าของตรวจ

### สิ่งที่ต้องระวัง

- **confidence < 80 = ห้าม approve เอง** ต้องส่งให้เจ้าของเสมอ
- **IG/TikTok ต้องมีรูป** — ถ้ายังไม่มี image_url อย่าเพิ่ง approve
- **อย่าแก้ Sheet ขณะที่ n8n กำลัง sync** (หลังเที่ยงคืน)

## 10. การแก้ปัญหา

### ปัญหาที่พบบ่อย

#### n8n workflow หยุดทำงาน
1. เปิด n8n UI → Workflows → ดูว่า Active ไหม
2. ถ้า Inactive → กด Activate
3. ถ้า Error → ดู Execution log → แก้ไขตามข้อผิดพลาด

#### Social post failed
1. เปิด n8n Executions → หา execution ที่ fail
2. ดู error message:
   - `401 Unauthorized` → token expired → ดูหัวข้อ Token หมดอายุ
   - `400 Bad Request` → content ไม่ถูก format (เช่น IG ไม่มีรูป)
   - `429 Too Many Requests` → rate limit → รอ 1 ชม. แล้วลองใหม่
3. แก้ปัญหาแล้ว → เปลี่ยน status กลับเป็น 'scheduled' ใน DB

#### Blog ไม่แสดงบนเว็บ
1. เช็ค status ใน DB → ต้องเป็น 'published'
2. เช็ค product_id → ต้องตรงกับ URL (เช่น 'customsguard')
3. เช็ค content_type → ต้องเป็น 'blog'

#### Google Sheets ไม่ sync
1. เช็ค n8n credentials → Google Sheets OAuth ยังใช้ได้ไหม?
2. เช็คว่า Sheet ID ถูกต้องใน n8n workflow
3. ลอง manual trigger workflow ใน n8n

## 11. กฎเหล็ก (ห้ามฝ่าฝืน)

1. **ห้าม Auto-Pilot 100%** — ทุกบทความต้องผ่าน human review + fact-check
2. **เลขพิกัด HS Code ต้องตรวจทุกครั้ง** — อ้างผิด = ลูกค้าโดนปรับ = แบรนด์พัง
   - **วิธีตรวจ:** กด `customs_verify_url` ในแต่ละ row → เทียบเลข + คำอธิบาย + อัตราภาษี กับเว็บกรมศุลฯ
   - **ขั้นตอนละเอียด:** ดูหัวข้อ [8. วิธี Review > วิธีตรวจเลข HS Code](#8-วิธี-review-ใน-google-sheets)
   - confidence < 80 → **ห้าม approve เอง** ส่งให้เจ้าของเสมอ
   - confidence < 80 + ยังไม่ fact-check → n8n **auto-reject** ห้าม schedule
3. **ต้องมีแหล่งอ้างอิง** — references_urls ทุกบทความ (กรมศุลกากร, ราชกิจจาฯ)
4. **Expert Opinion — AI เขียนให้แล้ว** ไม่ต้องเขียนเอง
   - AI Agent 2 (Grumpy Expert) เขียน "เรื่องจริงจากหน้าด่าน" + คำเตือน ใส่ในบทความให้แล้ว
   - หน้าที่พนักงาน: อ่านแล้วดูว่าสมเหตุสมผลไหม ถ้าไม่แน่ใจ → ส่งให้เจ้าของตรวจ
5. **ห้ามลง content พร้อมกันเยอะ** — ทำตามปฏิทินในหัวข้อ [5. งานประจำสัปดาห์](#5-งานประจำสัปดาห์) เท่านั้น (สัปดาห์ละ 1 ชุด)
6. **IG/TikTok ต้องมีรูปเสมอ** — โพสไม่มีรูป = fail
7. **ข้อมูลต้องเป็นปีปัจจุบัน** — FTA/ภาษี/กฎหมาย เปลี่ยนทุกปี
8. **Token refresh proactive** — n8n ตรวจทุกวัน ไม่ใช่รอหมดอายุแล้วค่อยแก้
