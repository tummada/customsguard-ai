# FAQ — คำตอบที่ต้องเตรียมไว้สำหรับชิปปิ้ง

---

## 1. "ข้อมูลลูกค้าผมหลุดไหม?"

> "ระบบแยกข้อมูลขาดจากกันด้วย Row-Level Security — ข้อมูลบริษัท A มองไม่เห็นบริษัท B เลยครับ AI ก็ไม่เอาไปเรียนรู้ข้ามบริษัท ข้อมูลเข้ารหัส HTTPS ทั้งขาส่ง และเก็บในฐานข้อมูลที่เข้ารหัสด้วยครับ"

**เบื้องหลัง:** Multi-tenant RLS (Row-Level Security) ทุก table มี tenant_id + PostgreSQL policy กรองอัตโนมัติ

---

## 2. "AI ผิดแล้วใครรับผิดชอบ?"

> "AI เป็นแค่ผู้ช่วยเสนอ draft ครับ ปุ่มยืนยันอยู่ในมือพี่ พี่ตรวจก่อนทุกครั้ง เหมือนเครื่องคิดเลข — ช่วยคำนวณ แต่คนกดปุ่มยังรับผิดชอบ"

**จุดสำคัญ:** ย้ำ Human-in-the-loop เสมอ ไม่ auto-submit อะไรทั้งนั้น

---

## 3. "IT ที่บริษัทห้ามลง Extension"

> "เข้าใจครับ ผมมีทางเลือกให้: เจน XML/EDI ให้ Import เข้าโปรแกรมที่พี่ใช้อยู่ ไม่ต้องลง Extension ก็ใช้ได้ครับ"

**เบื้องหลัง:** ทำ XML export เป็น fallback — ถ้าลง Extension ไม่ได้ ก็ยังใช้ผ่าน web หรือ API แล้ว export XML ได้

---

## 4. "มี GRI อ้างอิงมั้ย?"

> "ตอนนี้ยังไม่มีครับ แต่จดไว้แล้ว ถ้าพี่ต้องการจะเพิ่มให้ — พี่ว่า GRI ข้อไหนที่ใช้บ่อยที่สุดครับ?"

**เทคนิค:** ดึง requirement ต่อ — ถามว่า GRI Rule 1, 3a, 3b, 6 ข้อไหนใช้บ่อย

---

## 5. "เชื่อมต่อ Netbay/TIFFA ได้มั้ย?"

> "ตอนนี้ยังเชื่อมตรงไม่ได้ แต่ถ้าเจนเป็น XML มาตรฐานกรมฯ ให้ Import ได้ พี่จะสนใจมั้ยครับ? — พี่ใช้ XML schema ตัวไหนอยู่ครับ?"

**เทคนิค:** ดึง requirement เรื่อง XML schema + format ที่ใช้จริง

---

## 6. "ข้อมูลเก็บที่ไหน ใครเข้าถึงได้บ้าง?"

> "เก็บบน Cloud server ที่ผมดูแลเอง ไม่ได้ใช้ third-party AI เก็บข้อมูล ข้อมูล Invoice ของพี่ไม่ถูกส่งไปเทรน AI model ครับ — เฉพาะตอนสแกนเท่านั้นที่ส่งไป Gemini API แล้วลบทิ้งทันที"

**เบื้องหลัง:** Gemini API ไม่เก็บ data สำหรับ training (ตาม Google Cloud Terms) + เราเก็บแค่ผลลัพธ์ ไม่เก็บ raw PDF บน cloud นาน

---

## 7. "ราคาเท่าไหร่?" (ถ้าถูกถาม)

> "ตอนนี้ยังอยู่ในช่วงพัฒนาครับ ผมอยากรู้ก่อนว่าพี่ใช้งานจริงแล้วรู้สึกยังไง — ถ้าได้ลองใช้แล้วพี่คิดว่าคุ้มเท่าไหร่ครับ?"

**เทคนิค:** ไม่ตอบราคาตอนนี้ ดึง pricing signal จากชิปปิ้งก่อน

---

## 8. "ระบบ offline ใช้ได้มั้ย?" (ถ้าถูกถาม)

> "ข้อมูลที่เคยค้นไว้แล้วจะ cache อยู่ในเครื่องครับ ถ้า internet ช้าก็ยังดูผลลัพธ์เก่าได้ แต่ถ้าจะค้นใหม่ต้องมี internet ครับ ส่วน Draft ที่กำลังทำอยู่เซฟในเครื่องอัตโนมัติ ไม่หาย"

**เบื้องหลัง:** Dexie/IndexedDB cache สำหรับ HS Code + FTA lookup (12-24h TTL)

---

## 9. "ทำไมต้องใช้ Chrome Extension?" (ถ้าถูกถาม)

> "เพราะพี่เปิด e-Customs อยู่บน Chrome อยู่แล้วครับ Extension ทำงานอยู่ข้างๆ เลย ไม่ต้องสลับหน้าจอไปมา — เหมือนมีผู้ช่วยนั่งอยู่ข้างๆ คอยบอกข้อมูลให้ตลอด"

---

## 10. "ถ้า AI ตีพิกัดผิดแล้วไม่มีใครจับได้ล่ะ?" (คำถามยาก)

> "ระบบออกแบบให้มีไฟสัญญาณครับ — ถ้า AI ไม่มั่นใจจะขึ้นไฟเหลืองหรือแดง เตือนให้พี่ตรวจสอบเพิ่มเติม และทุกรายการ พี่ต้องกดยืนยันเองก่อนถึงจะส่งได้ ไม่มี auto-submit ครับ"

> "แถมอนาคตจะมี Audit Trail เก็บ log ว่าอ้างอิงจากเอกสารไหน พี่จะตรวจสอบย้อนหลังได้ทุกรายการ"
