# สคริปต์การนำเสนอ VOLLOS CustomsGuard (ไม่เกิน 20 นาที)

---

## 0. หมัดฮุก 30 วินาที — The Elevator Pitch

> **"พี่ครับ ผมไม่ได้มาขายโปรแกรมใหม่ แต่ผมเอา 'ผู้ช่วยอาวุโส' ที่จำพิกัดได้แม่น อ่าน PDF ได้เอง และคำนวณเงินภาษีที่พี่จะประหยัดได้ให้ใน 2 นาที มาฝากครับ"**

เทคนิค:
1. "ไม่ได้มาขาย" → ลดการ์ด
2. "ผู้ช่วยอาวุโส" → ไม่ใช่ AI มาแทน แต่เป็นลูกน้องเก่ง
3. "ประหยัดเงินภาษีใน 2 นาที" → Hook ด้วยเงิน

(วางตาราง "ก่อน vs หลัง" บนโต๊ะ)

---

## 1. เปิดหัว (2 นาที) — เชื่อมความเจ็บปวด

> "พี่ครับ ผมอยากเรียนถามก่อนว่า ตอนนี้วันๆ หนึ่ง **ใช้เวลาค้น HS Code กี่นาทีต่อรายการ** ครับ?"

(ฟัง → จดตัวเลข)

> "แล้วเคยเจอเคสที่ **ตีพิกัดผิด** แล้วต้องแก้ไขมั้ยครับ? เดือนหนึ่งเสีย **ค่า Amendment** เยอะไหม?"

(ฟัง → จดตัวเลข — นี่คือ ROI ที่สำคัญมาก)

> "ให้ผมโชว์ให้ดูเลยนะครับว่าผู้ช่วยตัวนี้ทำอะไรได้บ้าง"

---

## 2. Demo สด (12 นาที) — โชว์ 4 ฟีเจอร์หลัก

### ฟีเจอร์ 1: สแกน PDF อัตโนมัติ (3 นาที)

**Action:** เปิด Chrome Extension → ลาก PDF ใบขน/Invoice วาง

> "พี่ลองดูนะครับ ผมอัปโหลด PDF ใบขน → ระบบดึงรายการสินค้าออกมาให้อัตโนมัติเป็น draft ตรงไหนไม่มั่นใจจะขึ้นเป็นไฟเหลือง/แดง พี่ตรวจแก้ได้เลยก่อนกดยืนยัน"

> **"การตัดสินใจสุดท้ายอยู่ที่พี่เสมอ"**

> "แถมข้อมูลเซฟไว้ในเครื่องพี่อัตโนมัติ ถ้า browser ค้างก็ไม่หายครับ"

Key points:
- ดึง: HS Code, รายละเอียดสินค้า, จำนวน, น้ำหนัก, ราคา CIF
- รองรับทั้ง PDF ข้อความ + PDF สแกน (OCR)
- ไฟสัญญาณ เขียว/เหลือง/แดง = ระดับความมั่นใจ
- แก้ไขได้ทุกช่อง + Auto-Save Draft

---

### ฟีเจอร์ 2: ค้น HS Code ด้วยภาษาธรรมชาติ (3 นาที)

**Action:** เปิด Chat Panel → พิมพ์ "ผ้าฝ้ายทอมือ" หรือ "frozen shrimp"

> "สมมติพี่ไม่แน่ใจว่าสินค้าตัวนี้อยู่พิกัดไหน พิมพ์ว่า 'ผ้าฝ้ายทอมือ' → ระบบเรียงลำดับพิกัดที่น่าจะใช่ที่สุดให้เลย ไม่ต้องนั่งไล่ดูทั้งเล่ม"

> **"แต่พี่เลือกเองว่าจะใช้พิกัดไหน"**

**ถ้าถูกถาม "อ้างอิง GRI ข้อไหน?":**
> "ตอนนี้ระบบยังไม่ได้บอก GRI Rule ครับ แต่ผมจดไว้แล้ว ถ้าระบบบอกได้ว่าตีพิกัดตาม GRI Rule 1 หรือ 3a พี่จะสนใจมั้ยครับ?"

(= ดึง requirement ไปในตัว)

---

### ฟีเจอร์ 3: เตือน FTA + คำนวณเงินประหยัด (3 นาที) ★ ฟีเจอร์ขายเงิน

**Action:** เลือก HS Code → ดู FTA panel ที่แสดงส่วนต่างภาษี

> "สมมติพี่นำเข้าสินค้าจากญี่ปุ่น มูลค่า 1 ล้านบาท"

(ชี้หน้าจอ)

> "อัตราปกติ 10% = ภาษี 100,000 บาท"
>
> "แต่ระบบเจอว่าถ้าใช้ JTEPA ลดเหลือ 3.5% = ภาษี 35,000 บาท"
>
> **"ประหยัดได้ 65,000 บาทต่อ shipment"** — ระบบคำนวณให้เลย พร้อมบอกว่าต้องใช้ Form ไหน"

(ชี้สูตรในกระดาษ) **Savings = มูลค่า x (อัตราปกติ - อัตรา FTA)**

**เน้นตัวเลขเงินจริง** — นี่คือสิ่งที่เจ้าของบริษัทชิปปิ้งจะยอมจ่าย

---

### ฟีเจอร์ 4: ถาม-ตอบกฎระเบียบ + Audit Trail (3 นาที)

**Action:** เปิด Chat Panel → พิมพ์ "นำเข้าแบตเตอรี่ลิเธียม ต้องขอ อย. มั้ย?"

> "อันนี้กำลังพัฒนาอยู่ครับ ฟีเจอร์นี้จะช่วยค้นประกาศกรมศุลกากร + กฎระเบียบได้เลย ไม่ต้องเปิดหาเอง แถมยัง **เก็บ log ไว้ว่าอ้างอิงจากเอกสารไหน** ถ้าโดนตรวจย้อนหลังก็มีหลักฐานอ้างอิงได้"

เรื่องเรท:

> **"พี่ทราบดีว่าใช้เรทธนาคารคีย์งานคือจุดเริ่มต้นของใบแก้ ระบบของผมดึงเรทประกาศกรมศุลฯ รายสัปดาห์มาล็อคให้เลย พี่ไม่ต้องกังวลเรื่องนี้อีกต่อไปครับ"**

---

## 3. ปิดท้าย (5 นาที) — ดึง Requirement

> "พี่ครับ ที่โชว์ไปนี่ **ตรงไหนที่พี่คิดว่าจะช่วยงานได้จริง** ครับ?"

(จด → ถามต่อ)

> "แล้ว **ตรงไหนที่ยังขาด** หรือ **อยากให้มีเพิ่ม** ครับ?"

(จด → ถามต่อ)

> "ถ้าระบบสามารถ **เจนไฟล์ XML ตามมาตรฐานกรมศุลฯ** ให้พี่เอาไป Import เข้าโปรแกรมที่ใช้อยู่ได้เลย **พี่จะสนใจไหมครับ?**"

(ถ้าตาสว่าง → นี่คือ killer feature)

> "แล้วเวลา **ระบบ e-Customs ช้าหรือ iFrame ค้าง** พี่ทำยังไงครับ? ถ้า Extension มีระบบ **Auto-Save Draft** เก็บไว้ในเครื่องก่อนส่ง พี่ว่าดีมั้ย?"

(ถ้าพยักหน้า → แสดงว่าเข้าใจปัญหาจริง)

## ข้อควรระวังตลอดการนำเสนอ

| ห้ามพูด | พูดแทนว่า |
|---|---|
| "ระบบทำแทนได้" | "ระบบช่วยเตรียม draft ให้" |
| "100% ถูกต้อง" | "ไฟเขียว = มั่นใจสูง แต่ตัดสินใจสุดท้ายอยู่ที่พี่" |
| ดูถูกวิธีเดิม | "ผมรู้ว่าพี่ทำงานได้ดีอยู่แล้ว แค่อยากช่วยให้เร็วขึ้น" |

---

## Mindset ตลอดการนำเสนอ

- ระบบคือ **"Co-pilot"** ไม่ใช่ Autopilot
- ชิปปิ้ง **กลัว "ผิด" มากกว่ากลัว "ช้า"** เพราะสำแดงเท็จมีโทษทางกฎหมาย
- ลด 2 แรงต้าน: (1) กลัวตกงาน (2) กลัวรับผิดชอบทางกฎหมายถ้า AI ผิด
- **คำพูดที่ดี:** "AI ทำ draft ให้ แต่ปุ่มยืนยันอยู่ในมือพี่เสมอ"
