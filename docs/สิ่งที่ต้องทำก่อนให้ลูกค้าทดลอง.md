## Phase B: เตรียมให้คนลอง (1 วัน) — เจ้าของทำเอง

- [ ] 6. **เตรียม PDF ตัวอย่าง 2-3 ไฟล์** — invoice นำเข้าจริง (ลบข้อมูลลับก่อน)
- [ ] 7. **เขียนคำแนะนำใช้งาน 1 หน้า** — วิธีติดตั้ง Chrome Extension + ขั้นตอนใช้งาน
- [ ] 8. **สร้างช่องทางรับ feedback** — LINE OA หรือ Google Form (ถามว่าชอบอะไร/ไม่ชอบอะไร/จ่ายเท่าไหร่)
- [ ] 9. **หาคนลองใช้ 3-5 คน** — เพื่อน/คนรู้จักที่ทำ import-export ก่อน

## Phase C: ให้ลองใช้ + ฟัง feedback (1-2 สัปดาห์)

- [ ] 10. **นั่งข้างๆ ดูเขาใช้** — หรือ video call แชร์หน้าจอ ห้ามสอน ให้ดูว่าเขาติดตรงไหน
- [ ] 11. **จดทุกอย่าง** — สับสนตรงไหน, กดผิดตรงไหน, ต้องการอะไรที่ไม่มี
- [ ] 12. **ถามคำถามสำคัญ:**
  - ตอนนี้หา HS code ยังไง? ใช้เวลานานแค่ไหน?
  - เคยใส่ HS code ผิดไหม? เสียเงินเท่าไหร่?
  - เครื่องมือนี้สะดวกกว่าที่ทำอยู่ไหม?
  - จ่ายเดือนละเท่าไหร่?
- [ ] 13. **ตัดสินใจ** — ไปต่อ / pivot / หยุด

## หาลูกค้าทดลองได้ที่ไหน?

| ช่องทาง | วิธี |
|---------|------|
| คนรอบตัว | เพื่อน/ญาติที่ทำ import-export — ง่ายที่สุด ขอ 30 นาที |
| Facebook Group | "ชิปปิ้ง", "นำเข้า-ส่งออก", "Logistics Thailand" — โพสต์ขอคนทดลองฟรี |
| สมาคมชิปปิ้งแห่งประเทศไทย | ติดต่อขอนำเสนอเครื่องมือให้สมาชิกลอง |
| LinkedIn | ค้นหา "customs broker", "freight forwarder" ในไทย — DM ตรง |
| หอการค้าจังหวัด | มี SME ผู้นำเข้า-ส่งออกเป็นสมาชิก |

## สิ่งที่ยังไม่ต้องทำตอนนี้ (ทำหลังได้ feedback แล้ว)

| รายการ | ทำเมื่อไหร่ |
|--------|------------|
| ระบบ Payment / Subscription | มีคนพร้อมจ่ายจริงแล้ว |
| Usage Quota (จำกัดครั้ง) | มี paying customer แล้ว |
| Content Marketing / SEO | ได้ product-market fit แล้ว |
| Data Pipeline เพิ่ม (BOI, Excise) | ลูกค้าบอกว่าต้องการ |
| จ้างนักการตลาด | มี 5-10 คนที่ลองใช้แล้วพร้อมจ่าย |
