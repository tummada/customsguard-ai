  วิธีตรวจสอบคุณภาพข้อมูลเก่า vs ใหม่

  1. Spot-check กับแหล่งอ้างอิง (วิธีตรงที่สุด)

  - เลือก 10-20 HS codes ที่รู้คำตอบอยู่แล้ว (เช่น กุ้ง 0306.17, ข้าว 1006.30)
  - เทียบ rate ใน DB กับ rate ที่เปิดดูจากเว็บ thailandntr.com / customs.go.th ตรงๆ
  - ถ้าไม่ตรง → ข้อมูลมีปัญหา

  2. Row count + coverage check

  - เก่ามี 19,689 HS codes → ใหม่ต้องได้ใกล้เคียงหรือมากกว่า
  - FTA rates เก่ามีแค่ 6 demo codes → ใหม่ต้องมี 12,000+ × 10 FTAs
  - ถ้าน้อยกว่า = โหลดไม่ครบ

  3. Semantic search quality (ทดสอบ RAG)

  - ใช้ 11_quality_check.py ที่มีอยู่แล้ว — มี 20 test queries ภาษาไทย+อังกฤษ
  - ยิง query เดียวกันใส่ข้อมูลเก่า vs ใหม่ → เปรียบเทียบ top-5 results
  - ข้อมูลดี = ค้นเจอ HS code ที่ตรงประเด็น

  4. Format validation (อัตโนมัติ)

  - utils/validator.py มีอยู่แล้ว — ตรวจ HS code format (\d{4}\.\d{2}\.\d{2}), rate range (0-80%), FTA name ถูกต้อง
  - รันทั้ง 2 ชุด เทียบจำนวน invalid records

  5. วิธีที่ผมแนะนำจริงๆ

  ไม่ต้องทำใหม่ทั้งหมด — ทำแบบนี้ดีกว่า:

  1. HS codes 19,689 ตัว → เก็บไว้ เพราะมาจาก data.go.th (open data ราชการ) น่าเชื่อถือ
  2. FTA JSON 10 ไฟล์ → spot-check 10-20 codes ก่อน ถ้า OK ก็ bulk insert เข้า DB ได้เลย
  3. Regulations 10 ตัว → ต้องตรวจว่า AI สร้างหรือ copy จากของจริง ถ้า AI สร้าง → ทิ้ง แล้วเอาของจริงมาแทน

  สรุป: ไม่ต้องทำใหม่ทั้ง pipeline แต่ ตรวจสอบข้อมูลที่มี ก่อน ถ้าพบว่าผิดค่อยโหลดใหม่เฉพาะส่วนนั้น ประหยัดเวลาและ cost กว่ามาก
