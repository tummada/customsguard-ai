# อธิบายระบบ VOLLOS แบบเด็ก 5 ขวบเข้าใจ

## ร้านขายของเล่นวิเศษ

ลองนึกภาพว่าเรามี **ร้านขายของเล่นวิเศษ** อยู่ร้านนึง

ร้านนี้มีของที่ทุกคนต้องใช้เหมือนกัน:
- **ประตูหน้าร้าน** (ระบบล็อกอิน) → ต้องบอกชื่อก่อนถึงจะเข้าได้
- **กระปุกหยอดเหรียญ** (ระบบเงิน) → ต้องมีเหรียญถึงจะเล่นของเล่นได้
- **ป้ายชื่อ** (ระบบ Tenant) → เด็กแต่ละคนมีกล่องของตัวเอง จะไปแอบเปิดกล่องคนอื่นไม่ได้

---

## แล้วถ้าอยากเพิ่มของเล่นใหม่ล่ะ?

ตอนแรกร้านมี **มุมต่อเลโก้** (CustomsGuard - ระบบใบขน) อยู่มุมเดียว

แต่วันนึงเจ้าของร้านอยากเพิ่ม:
- **มุมหมอฟันจำลอง** (ระบบร้านหมอฟัน)
- **มุมวาดรูป** (ไอเดีย SaaS อื่นๆ ในอนาคต)

### ปัญหาคือ: จะเพิ่มมุมใหม่ยังไง โดยไม่ต้องรื้อร้านทั้งร้าน?

---

## วิธีแก้: ร้านแบบต่อเติมได้!

เราออกแบบร้านให้เป็น **ห้องๆ** ที่ถอดเข้าถอดออกได้ เหมือนตัวต่อเลโก้เลย!

```
🏠 ร้านของเล่นวิเศษ (platform-app)
│
├── 🧱 ฐานร้าน (platform-core)     ← ของที่ทุกห้องใช้ร่วมกัน
│   ├── ประตูหน้าร้าน
│   ├── กระปุกเหรียญ
│   └── ป้ายชื่อเด็กแต่ละคน
│
├── 🟦 ห้องต่อเลโก้ (feature-customsguard)    ← ถอดออกได้!
├── 🟩 ห้องหมอฟัน (feature-dental)             ← เสียบเข้าได้!
└── 🟨 ห้องวาดรูป (feature-xxx)                ← เพิ่มทีหลังได้!
```

### กติกาง่ายๆ:

1. **แต่ละห้องอยู่เป็นอิสระ** → ห้องหมอฟันพังก็ไม่กระทบห้องเลโก้
2. **เด็กต้องสมัครก่อน** → ถ้าไม่ได้สมัครห้องหมอฟัน ก็เข้าห้องนั้นไม่ได้
3. **เพิ่มห้องใหม่ง่ายมาก** → แค่สร้างห้องใหม่แล้วเสียบเข้าร้าน ไม่ต้องรื้ออะไรเลย
4. **กล่องของใครของมัน** → เด็กคนนึงจะไปเปิดกล่องของเด็กอีกคนไม่ได้ ไม่ว่าจะอยู่ห้องไหน

## สรุปสั้นๆ

| เปรียบเทียบ | ในระบบจริง |
|---|---|
| ร้านของเล่น | แพลตฟอร์ม VOLLOS |
| ฐานร้าน | platform-core (ล็อกอิน, จ่ายเงิน, แยกข้อมูล) |
| ห้องต่อเลโก้ | feature-customsguard (ระบบใบขน) |
| ห้องหมอฟัน | feature-dental (ระบบร้านหมอฟัน) |
| เด็กแต่ละคน | Tenant (บริษัทแต่ละบริษัท) |
| กล่องของเล่นส่วนตัว | Row-Level Security (ข้อมูลแยกกัน) |
| บัตรสมาชิกห้อง | tenant_features (สมัครใช้ feature ไหนบ้าง) |
| เพิ่มห้องใหม่ | สร้าง feature-xxx แล้วเสียบเข้าระบบ |
